Materi tentang Version Control Git


Version control adalah sebuah sistem yang merekam perubahan-perubahan dari sebuah berkas atau sekumpulan berkas dari waktu ke waktu sehingga Anda dapat menilik kembali versi khusus suatu saat nanti. Sebagai contoh, pada buku ini Anda akan menggunakan sumber kode perangkat lunak sebagai berkas-berkas yang direkam dengan version control, walau pada kenyataannya Anda dapat melakukan ini dengan hampir semua jenis berkas pada komputer.



Local Version Control System


Metode version-control yang digunakan oleh banyak orang adalah menyalin dokumen ke folder lain. Pendekatan ini sangat umum karena sangat sederhana, tetapi juga sangat rentan terjadi kesalahan. Sangatlah mudah untuk lupa folder mana yang sedang digunakan dan secara tidak sengaja menulis ke dokumen yang salah atau menyalin dokumen yang tidak dimaksudkan. Untuk menyelesaikan masalah ini,


Centralized Version Control System


Masalah besar lainnya yang orang-orang temui adalah mereka perlu berkolaborasi untuk mengembangkan sebuah sistem. Untuk menangani masalah ini, Centralized Version Control System dikembangkan. Sistem ini memiliki sebuah server tunggal yang berisi seluruh versi dari dokumen, dan sejumlah orang yang mengunduh dokumen dari tersebut.


Distributed Version Control System (DVCS)


Dalam DVCS (seperti Git, Mercurial, atau Darcs), klien tidak hanya mengunduh salinan terbaru dari dokumen; melainkan, menyalin seluruh repositori, termasuk riwayat lengkapnya. Oleh karena itu, jika server mati dan sistem-sistem ini berkolaborasi melalui server tersebut


GIT

Git adalah distributed version control software system yang mampu menangani perubahan dari source code atau data. Git sering digunakan untuk mengontrol source code oleh programmers yang secara berkolaborasi mengembangkan sistem.

Fungsi Fungsi dari Perintah
  • git init = untuk membuat repository kosong yang baru
  • git add . = untuk menandai seluruh perubahan yang terjadi pada repo
  • git status = digunakan untuk mengecek daftar perubahan yang terjadi
  • git commit = untuk menyimpan seluruh perubahan yang ada pada staging area
  • git log = untuk menampilkan daftar commit yang telah dilakukan pada repository
  • git checkout = untuk beralih ke branch atau commit tertentu
  • git branch -a = untuk melihat branch yang ada pada repository
  • git branch namabranch = Untuk membuat branch baru
  • git push origin = untuk mengirim perubahan berupa commit yang dilakukan ke remote repository
  • git remote -v = Untuk mengecek remote repository yang terhubung
